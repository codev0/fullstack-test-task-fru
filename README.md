# Тестовое на вакансию Fullstack Node.js Developer

Сразу решил делать по бонус треку, тк ранее уже делал MVP виджета который встраивался в страницу на Preact и было базовое понимание принципа работы.

Трудности:

1. Организовать так чтобы было удобно запустить демо, бэк, и трекер вместе.
2. Тесты для трекера, работа с таймаутами заставила мозги поработать, не уверен что до конца понял как это работает.

Временные затраты 25 часов примерно.

1. 5 часов чтобы разобраться в трекерах и фастифай.
2. 10 часов ушло чтобы организовать код, сборку, и причесать все. Я ранее не делал приложения где фронт и бэк собирались вместе.
3. 10 часов на имплементацию и тесты.

Запуск демо:

1. Установите зависимости `npm ci`
2. Создайте файл `.env`
3. Скопируйте содержимое `.env.example` в `.env`
4. Поставьте нужные вам значения в `.env`
5. Далее запустите: `npm run demo`

Работает через сниппет тк это более надежное решение:

1. не блокирует поток отрисовки тк трекер подгружается асинхронно
2. собирает события раньше насколько это возможно
3. после загрузки (IIFE) и инциации трекера все события накопленные во временной очереди будут обработаны

## Структура папок

1. `public` для раздачи статики трекера
2. `demo` для проверки трекера в браузере через пакет serve
3. `src` серверный код
4. `tracker` код трекера который билдится в паблик и раздается сервером из папки public
5. `tests` тесты для бэкенда

## Код бэкенда

```
├── src
│   ├── plugins // настроенные плагины фастифай для автолоудера.
│   ├── routes // контроллеры.
│   ├── apps // бизнес код.
│   ├── app.ts // инициализируем плагины, роуты, и переопределяем дефолты.
│   └── server.ts // точка входа в бэкенд. инициируем апку, подбрасываем в сервер, запускаем сервер.
```
